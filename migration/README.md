# Миграция данных из Google Sheets

Пошаговый перенос клиентской базы из Google Sheets в LMS.

---

## Шаг 1 — Подготовка Google API

1. Перейди в [Google Cloud Console](https://console.cloud.google.com/)
2. Создай проект (или выбери существующий)
3. Включи API: **Google Sheets API** + **Google Drive API**
4. Создай **Service Account**: IAM и администрирование → Сервисные аккаунты → Создать
5. Скачай ключ в формате **JSON** → сохрани как `migration/service_account.json`
6. **Открой доступ к таблице**: в Google Sheets → Поделиться → введи email сервисного аккаунта (из JSON-файла поле `client_email`) с правом **Читатель**

---

## Шаг 2 — Настройка переменных

```bash
cp migration/.env.example migration/.env
# Открой migration/.env и заполни:
# GOOGLE_SPREADSHEET_ID=...  (из URL таблицы)
# DATABASE_URL=...
```

---

## Шаг 3 — Установка зависимостей

```bash
make install-migration
```

---

## Шаг 4 — Запуск миграции (по шагам)

```bash
# 1. Выгрузить данные из Google Sheets → migration/raw_data/*.json
make migration-extract

# 2. Нормализовать и дедуплицировать → migration/clean_data/clean.json
make migration-transform

# 3. Загрузить в PostgreSQL
make migration-load

# 4. Верификация — проверить балансы, статусы, отсутствие дублей
make migration-verify
```

---

## Структура таблицы

Скрипт автоматически ищет листы по имени:
- **Клиенты**: лист содержит "клиент" или "client" в названии
- **Сделки**: лист содержит "сделк", "deal" или "займ" в названии

### Ожидаемые колонки листа клиентов

| Колонка в Sheets | Описание |
|---|---|
| `ФИО` или `full_name` | Полное имя клиента |
| `Телефон` или `phone` | Номер телефона (любой формат) |
| `Паспорт` или `passport` | Серия и номер паспорта (необязательно) |
| `Адрес` или `address` | Адрес (необязательно) |

> Если колонки называются иначе — отредактируй `transform.py`, функцию `_find_clients_sheet()`.

---

## Откат при ошибке

```bash
# Удаляет все импортированные данные (клиентов, сделки, платежи)
# Требует подтверждения
DATABASE_URL=... python3 migration/rollback.py --confirm
```

---

## Параллельная работа (рекомендуется)

По плану: **2 недели** работать одновременно в Sheets и LMS, затем переключиться.

1. Запусти миграцию → проверь данные в LMS
2. В течение 2 недель новые записи вводи **в оба места**
3. После перехода: Sheets переведи в режим "только чтение" → архив
